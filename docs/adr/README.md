# ADR

Architecture Decision Records are required from day one.

Every important decision gets its own Markdown file.

## Format

Use this structure for each ADR:

- `ADR-0001`
- `Title`
- `Status`
- `Context`
- `Decision`
- `Consequences`
- `Alternatives Considered`
- `Date`

## Purpose

ADRs are the project memory for major technical, product, and architectural decisions.
They make the reasoning behind the system durable, searchable, and reviewable over time.
